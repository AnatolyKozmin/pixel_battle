# Разрешение конфликта git

## Текущая ситуация

У вас конфликт в `docker-compose.ip.yml`. Файл уже исправлен - убран дубликат строки.

## Что нужно сделать на сервере:

```bash
# 1. Проверить, что конфликт разрешен
git status

# 2. Добавить разрешенный файл
git add docker-compose.ip.yml

# 3. Удалить резервную копию (если есть)
rm docker-compose.ip.yml.bak

# 4. Очистить stash (если больше не нужен)
git stash drop

# 5. Проверить статус
git status
```

## Если нужно сохранить ваш IP (89.104.71.34):

Вместо того чтобы хранить IP в git, лучше использовать переменные окружения:

```bash
# Создайте .env файл или используйте переменные окружения
export SERVER_IP=89.104.71.34

# Или используйте скрипт для настройки
./scripts/setup-ip.sh 89.104.71.34
```

Это позволит:
- Не хранить IP в git
- Легко менять IP для разных серверов
- Избегать конфликтов при обновлениях
